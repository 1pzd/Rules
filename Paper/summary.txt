论文一：Trace2Skill — 将轨迹局部经验蒸馏为可迁移的 Agent 技能
================================================================================
全称: Trace2Skill: Distill Trajectory-Local Lessons into Transferable Agent Skills
作者: Jingwei Ni, Yihao Liu, Xinpeng Liu 等 (ETH Zurich / 北大 / 阿里)
链接: https://arxiv.org/abs/2603.25158

--------------------------------------------------------------------------------
【目的】
--------------------------------------------------------------------------------
解决 LLM Agent 技能（Skill）的自动化生成与优化问题。现有技能要么依赖人工编写
（不可扩展），要么纯靠模型参数化知识生成（容易遗漏关键操作陷阱）。Trace2Skill
旨在从 Agent 的实际执行轨迹中归纳提炼出可复用、可迁移的技能目录（Skill 
Directory），无需更新模型参数，也无需在推理时进行检索。

--------------------------------------------------------------------------------
【创新点】
--------------------------------------------------------------------------------
1. 并行多对一归纳整合（Parallel Many-to-One Consolidation）：
   - 与传统的"逐条轨迹顺序编辑"不同，Trace2Skill 先并行分析大量轨迹，再通过
     层次化合并将反复出现的经验教训压缩为一个统一技能。
   - 这是一种对 Agent 经验的归纳推理（Inductive Reasoning），类似于人类专家
     检查大量案例后撰写标准操作流程（SOP）。

2. 非对称分析师设计（Asymmetric Analyst Design）：
   - 成功分析师（𝒜+）：单次分析，从成功轨迹中提取可复用行为模式。
   - 失败分析师（𝒜-）：ReAct 风格循环，可检查文件产物、对比真值、验证修复
     方案后再提出补丁。无法解释因果的失败会被排除。

3. 技能可迁移性（Skill Portability）：
   - 进化出的技能不仅在源模型上有效，还能跨模型规模（35B→122B）、跨模型家族
     （Qwen→Gemma→GPT）、跨任务领域（电子表格→表格QA→数学推理→文档VQA）
     进行迁移。
   - 关键实验：从 Qwen3.5-35B 轨迹进化的技能可将 Qwen3.5-122B 在 
     WikiTableQuestions 上提升 57.65 个百分点。

4. 两种模式覆盖：
   - 技能深化（Deepening）：从人工编写的技能出发进行增强。
   - 技能创建（Creation）：从 LLM 参数化知识生成的弱草稿出发构建技能。

--------------------------------------------------------------------------------
【实验方法与过程】
--------------------------------------------------------------------------------
三阶段流水线：
  Stage 1 - 轨迹生成：使用冻结的 Agent（ReAct 风格）在进化集上运行，收集成功
            和失败轨迹。
  Stage 2 - 并行补丁提案：错误/成功分析师子 Agent 独立地从单条轨迹中提出技能
            补丁，形成补丁池。
  Stage 3 - 补丁整合：层次化合并（对数级别），去重、解决冲突、保留非重叠洞见，
            最终产出一个统一的技能更新。

评估领域：
  - 电子表格工作流：SpreadsheetBench（400样本，200进化/200测试），OOD迁移至
    WikiTableQuestions 和 HiTab。
  - 数学推理：400 DAPO 问题生成技能，100 DAPO + AIME 2026 测试。
  - 文档视觉问答（DocVQA）：50 例生成技能，5299 例测试。
  - 更广泛应用：PDF/PPTX/DOCX 编辑任务。

核心设计对比实验：
  - 并行整合 vs 顺序编辑：并行约3分钟完成，顺序编辑需15-60分钟，且并行质量更优。
  - 统一技能 vs 检索记忆（ReasoningBank）：统一技能持续优于检索式记忆。
  - Agentic 错误分析 vs 单次 LLM 调用：交互式诊断比仅读日志更精确。

--------------------------------------------------------------------------------
【模型】
--------------------------------------------------------------------------------
- 技能作者/用户：Qwen3.5-122B-A10B、Qwen3.5-35B-A3B（100% 自进化）
- 跨家族验证：Gemma-4-31B-it、GPT-5.5-high
- 核心特点：同一模型同时充当轨迹生成器、分析师和合并算子，无需外部进化/教师模型

--------------------------------------------------------------------------------
【Prompt 设计】
--------------------------------------------------------------------------------
论文在附录中提供了完整的 Prompt 模板：

1. Stage 1 Agent 系统提示：指导 ReAct 风格的工具使用和推理。

2. Stage 2 分析师提示：
   - 错误分析师提示（𝒜-）：要求检查轨迹和文件产物，对比真值输出，验证修复方案
     后才提出补丁。明确要求"无法因果解释的失败应被排除"。
   - 成功分析师提示（𝒜+）：识别成功轨迹中的可复用行为模式。
   - 输出格式：简洁、可操作的补丁，遵循技能编写指南。

3. Stage 3 合并算子提示（ℳ）：
   - 指示优先保留跨轨迹反复出现的模式（系统性特征），丢弃个别性编辑。
   - 去重、解决冲突、保留非重叠洞见。
   - 输出格式：diff 风格的编辑指令。

--------------------------------------------------------------------------------
【关键实验数据】
--------------------------------------------------------------------------------
- 35B 深化 +Combined 在 Avg 指标上取得最佳绝对分数。
- 从 Gemma-4-31B-it 轨迹进化的技能成功迁移到 Qwen3.5-122B。
- 122B 深化 +Combined：SpreadsheetBench Vrf 达 69.83（原 48.33），WikiTQ 达 79.24。
- DocVQA：Combined 设置下 ANLS 提升 0.2534，Acc 提升 22.25pp。


================================================================================
论文二：SkillClaw — 让技能在多用户生态中集体进化
================================================================================
全称: SkillClaw: Let Skills Evolve Collectively with Agentic Evolver
作者: Ziyu Ma, Shidong Yang, Yuxiang Ji 等
链接: https://arxiv.org/abs/2604.08377

--------------------------------------------------------------------------------
【目的】
--------------------------------------------------------------------------------
解决 LLM Agent（如 OpenClaw）技能在部署后静态不变的问题。不同用户的交互提供了
互补信号（当技能何时有效/失败），但现有系统缺乏将这些异质经验转化为可靠技能更
新的机制。SkillClaw 旨在实现跨用户的集体技能进化，让一个用户上下文中发现的改进
能传播到整个系统。

--------------------------------------------------------------------------------
【创新点】
--------------------------------------------------------------------------------
1. 集体技能进化（Collective Skill Evolution）：
   - 将多用户、跨时间的交互轨迹作为技能改进的主要信号来源。
   - 技能在共享仓库中维护，跨用户同步，无需用户额外操作。

2. 自主进化器（Autonomous Evolver）：
   - 持续聚合使用过程中生成的轨迹。
   - 自主识别反复出现的行为模式，将其转化为技能更新（优化现有技能或扩展新能力）。

3. 跨用户知识迁移：
   - 一个用户上下文中发现的改进可系统级传播。
   - 无需用户主动参与技能维护。

4. 与 OpenClaw 生态的结合：
   - 专注于真实世界 Agent 场景（WildClawBench）。

--------------------------------------------------------------------------------
【实验方法与过程】
--------------------------------------------------------------------------------
- 评估基准：WildClawBench（真实世界 Agent 场景）
- 基座模型：Qwen3-Max
- 流程：多用户使用 → 轨迹聚合 → 进化器识别模式 → 技能更新 → 跨用户同步
- 在有限交互和反馈条件下显著提升性能

--------------------------------------------------------------------------------
【模型】
--------------------------------------------------------------------------------
- 基座 LLM：Qwen3-Max
- 评估环境：OpenClaw Agent 生态系统

--------------------------------------------------------------------------------
【Prompt 设计】
--------------------------------------------------------------------------------
论文为 work-in-progress 版本，Prompt 细节待后续版本补充。核心设计为自主进化器
的轨迹分析和技能更新逻辑。

--------------------------------------------------------------------------------
【关键实验数据】
--------------------------------------------------------------------------------
- 在 WildClawBench 上，有限交互和反馈下显著提升 Qwen3-Max 在真实世界 Agent 
  场景中的表现（具体数值待后续版本更新）。


================================================================================
论文三：CoEvoSkills — 通过协同进化验证实现技能自进化
================================================================================
全称: CoEvoSkills: Self-Evolving Agent Skills via Co-Evolutionary Verification
作者: Hanrong Zhang, Shicheng Fan, Henry Peng Zou 等 (UIC / MBZUAI / McGill 等)
链接: https://arxiv.org/abs/2604.01687

--------------------------------------------------------------------------------
【目的】
--------------------------------------------------------------------------------
使 Agent 能够自主生成复杂的多文件技能包（而非简单的工具函数或提示模板）。现有
自进化方法是为单一工具设计的，无法应对技能的多文件结构复杂性。同时，现有方法
过度依赖真值监督信号，在真实场景中不可用。

--------------------------------------------------------------------------------
【创新点】
--------------------------------------------------------------------------------
1. 协同进化框架（Co-Evolutionary Framework）：
   - Skill Generator：迭代生成和优化技能包，维护持久对话上下文。
   - Surrogate Verifier：信息隔离的独立 LLM 会话，综合测试用例和脚本，
     在无法访问真值测试内容的情况下提供高保真反馈。
   - 两者交替优化：当代理测试通过但真值甲骨文失败时，触发测试升级。

2. 信息隔离的代理验证（Informationally Isolated Surrogate Verification）：
   - 验证器完全独立于生成器的推理、代码和技能内容。
   - 仅观察任务指令和输出文件，防止确认偏差。
   - 确保验证器的测试生成与生成器内部状态条件独立。

3. POMDP 形式化：
   - 将技能生成环境形式化为部分可观察马尔可夫决策过程。
   - Agent 从未知道留出的真值测试检查什么，成功标准完全隐藏。

4. "Agent 能创建比人类更好的技能"：
   - 自进化技能编码了 Agent 原生的推理模式、工具使用偏好和分解策略。
   - 而非遵循人类关于 Agent 应该如何操作的假设。

--------------------------------------------------------------------------------
【实验方法与过程】
--------------------------------------------------------------------------------
评估基准：SkillsBench（87个任务，约20个专业领域，每个任务配确定性验证器）

基线对比：
  1. No-Skill：无技能基线（30.6%）
  2. Self-Generated Skills：一次性自生成（32.0%）
  3. CoT-Guided Self-Generation：CoT引导自生成（30.7%）
  4. Skill-Creator：Anthropic 官方技能创建器（34.1%）
  5. Human Curated Skills：人工策划技能（53.5%）
  6. CoEvoSkills：协同进化（71.1%，+40.5pp over no-skill）

进化动态：
  - Round 0（一次性生成无验证）：与 no-skill 持平
  - Round 2：44%
  - Round 3：超越人工策划技能（63%）
  - Round 5：收敛至 75%
  - 平均每任务需 4.1 次验证循环，2.4 次进化迭代

跨模型迁移：
  - Claude Opus 4.6 进化的技能迁移到 GPT-5.2、Claude Sonnet 4.5、Claude Haiku 4.5、
    Qwen3-Coder-480B、DeepSeek V3-671B、Mistral Large 3-675B
  - 所有模型获得 +36~44pp 提升

领域分析：
  - 11 个领域中 9 个超越人工策划技能
  - 金融（+56.9pp）、网络安全（+23.2pp）提升最大
  - 自然科学领域：人工策划技能反而降低性能，自进化技能大幅改善

--------------------------------------------------------------------------------
【模型】
--------------------------------------------------------------------------------
- 进化 Agent：Claude Opus 4.6、GPT-5.2
- 迁移验证：6 个额外模型（5 家公司）
- 评估框架：Claude-Code

--------------------------------------------------------------------------------
【Prompt 设计】
--------------------------------------------------------------------------------
论文提供了完整的 Prompt 模板（附录 F）：

1. Evolution Agent System Prompt：指导进化 Agent 的系统级指令。
2. Skill Discovery Hint：技能发现提示。
3. Skill-Creator Autonomous Mode Instruction：将 Anthropic 的交互式技能创建
   替换为自主模式——第一个会话迭代起草、自测、优化技能（至少3次迭代），
   第二个会话使用技能解决任务。
4. Self-Generated Skills Prompt：一次性自生成基线的提示。
5. CoT-Guided Self-Generation Prompt：五步结构化 CoT 提示。

核心设计：
  - 验证器生成代理测试套件 V = {e1, ..., e|V|}
  - 代理奖励 R~(x, V) = (1/|V|) * sum(1[e_k(x)])
  - 技能优化：S^(i+1) = argmax_S R~(Phi(S, E), V^(j))
  - 测试升级：V^(j+1) ~ pi^V_theta(·|I, x^(i), V^(j))

--------------------------------------------------------------------------------
【关键实验数据】
--------------------------------------------------------------------------------
- CoEvoSkills 达到 71.1% 通过率（+40.5pp over no-skill，+17.6pp over 人工策划）
- 跨模型迁移：GPT-5.2 受益于 Opus 进化的技能（65.0%），但其自进化技能（69.8%）
  仍优于迁移技能 4.8pp


================================================================================
论文四：SkillRL — 通过递归技能增强强化学习进化 Agent
================================================================================
全称: SkillRL: Evolving Agents via Recursive Skill-Augmented Reinforcement Learning
作者: Peng Xia, Jianwen Chen, Hanyang Wang 等 (多机构)
链接: https://arxiv.org/abs/2602.08234

--------------------------------------------------------------------------------
【目的】
--------------------------------------------------------------------------------
解决 LLM Agent 从过去经验中学习的困难。现有记忆方法主要存储原始轨迹，存在大量
冗余和噪声，无法提取高层次、可复用的行为模式。SkillRL 通过自动技能发现和递归
进化，弥合原始经验与策略改进之间的鸿沟。

--------------------------------------------------------------------------------
【创新点】
--------------------------------------------------------------------------------
1. 基于经验的技能蒸馏（Experience-Based Skill Distillation）：
   - 成功轨迹：提取战略模式、关键决策点、推理背后的正确行动。
   - 失败轨迹：合成简洁的失败教训——(1)失败点 (2)错误推理/行动 (3)应该怎么做
     (4)预防类似失败的一般原则。将冗长的失败片段转化为反事实。

2. 层次化技能库（Hierarchical SkillBank）：
   - 通用技能（S_g）：跨所有任务类型适用的普遍战略原则（探索策略、状态管理、
     目标追踪）。
   - 任务特定技能（S_k）：针对特定任务类别的专业知识（领域特定动作序列、
     前置条件、常见失败模式）。
   - 每个技能包含：名称、原则描述、适用条件。

3. 自适应检索策略：
   - 通用技能始终包含。
   - 任务特定技能通过语义相似度检索：TopK({s: sim(ed, es) > delta}, K)
   - Token 压缩：比原始轨迹压缩 10-20 倍。

4. 递归技能进化（Recursive Skill Evolution）：
   - 冷启动 SFT：教师模型生成技能增强推理轨迹，教基础模型如何检索和使用技能。
   - RL 训练中递归进化：每个验证 epoch 后，监控各类别成功率，仅对低于阈值的
     类别触发进化。
   - 多样性感知分层采样：按类别分组，按失败严重程度优先，轮询采样保持类别熵。
   - 形成良性循环：Agent 改进 → 遇到新挑战 → 驱动技能库扩展 → 进一步改进。

--------------------------------------------------------------------------------
【实验方法与过程】
--------------------------------------------------------------------------------
评估环境：
  - ALFWorld：基于文本的家务任务游戏
  - WebShop：模拟网络购物
  - 7 个搜索增强 QA 任务：NQ、TriviaQA、PopQA（单跳），HotpotQA、2Wiki、
    MuSiQue、Bamboogle（多跳）

基线对比：
  - 闭源 LLM：GPT-4o、Gemini-2.5-Pro
  - 提示/记忆方法：ReAct、Reflexion、Mem0、ExpeL、MemP、SimpleMem
  - RL 方法：RLOO、GRPO
  - 记忆增强 RL：MemRL、EvolveR、Mem0+GRPO、SimpleMem+GRPO

训练细节：
  - 基础模型：Qwen2.5-7B-Instruct
  - 教师模型：OpenAI o3
  - RL 算法：GRPO（学习率 1e-6，batch 16，group 8）
  - 任务特定技能检索：K=6，delta=0.4

消融实验：
  - 去除层次结构：ALFWorld -13.1%，WebShop -11.3%
  - 用原始轨迹替代技能库：最大降级（达 -25%）
  - 去除冷启动 SFT：-20%
  - 去除动态进化：-5.5%

技能库增长：
  - 初始：55 个技能（12 通用 + 43 任务特定）
  - 训练结束：100 个技能（20 通用 + 80 任务特定）

--------------------------------------------------------------------------------
【模型】
--------------------------------------------------------------------------------
- 基础模型：Qwen2.5-7B-Instruct
- 教师模型：OpenAI o3（用于技能蒸馏和 SFT 数据生成）
- RL 算法：GRPO

--------------------------------------------------------------------------------
【Prompt 设计】
--------------------------------------------------------------------------------
论文附录提供了详细 Prompt：

1. Agent 执行提示（Agent Execution Prompts）：指导 Agent 如何在环境中行动。
2. 技能生成和蒸馏提示（Skill Generation and Distillation Prompts）：
   - 成功轨迹蒸馏：识别关键决策点、正确行动背后的推理、可迁移的模式。
   - 失败轨迹蒸馏：分析失败点、错误推理、正确做法、预防原则。
3. 冷启动轨迹生成提示（Cold-start Trajectory Generation Prompts）：
   - 教师模型生成技能增强推理轨迹，展示如何检索、解读和应用技能。

技能格式：
  - 简洁名称（如"系统探索"）
  - 原则描述（策略说明）
  - when_to_apply 条件（适用场景）

--------------------------------------------------------------------------------
【关键实验数据】
--------------------------------------------------------------------------------
- ALFWorld：89.9% 成功率（超 GPT-4o 41.9pp，超 Gemini-2.5-Pro 29.6pp）
- WebShop：72.7% 成功率，85.2 分数
- 搜索增强 QA 平均：47.1%（超 Search-R1 的 38.5%，超 EvolveR 的 43.1%）
- Bamboogle（多跳）：73.8%（超 EvolveR 19.4pp）
- SkillRL 使用 GRPO 作为基础优化器，ALFWorld 上比 GRPO 绝对提升 12.3%
  （77.6% → 89.9%），归因于技能增强机制


================================================================================
论文五：Skill-SD — 面向多轮 LLM Agent 的技能条件自蒸馏
================================================================================
全称: Skill-SD: Skill-Conditioned Self-Distillation for Multi-turn LLM Agents
作者: Hao Wang, Guozhi Wang, Han Xiao 等 (vivo AI Lab / UCAS / CUHK)
链接: https://arxiv.org/abs/2604.10674

--------------------------------------------------------------------------------
【目的】
--------------------------------------------------------------------------------
解决强化学习训练 LLM Agent 面临的稀疏奖励和长 horizon 问题。现有的在策略自蒸馏
（OPSD）通过特权教师提供密集的 token 级监督，但固定的特权信息无法捕捉 Agent 任务
中多样化的有效策略，且简单地将 OPSD 与 RL 结合常导致训练崩溃。

--------------------------------------------------------------------------------
【创新点】
--------------------------------------------------------------------------------
1. 技能作为动态教师信号（Skill as Dynamic Teacher Signal）：
   - 将已完成的轨迹总结为紧凑的自然语言技能，描述成功行为、错误和工作流。
   - 技能仅条件化教师，学生始终在纯任务提示下行动。
   - 学生通过蒸馏学习内化技能指导，推理时不依赖检索。

2. 重要性加权反向 KL 损失（Importance-Weighted Reverse-KL Loss）：
   - 解决跨提示自蒸馏中的分布不匹配问题。
   - 学生和教师共享参数但条件于不同提示，朴素 k3 微分产生梯度偏差。
   - 推导出 rho * k3 损失，提供梯度校正的 token 级蒸馏。
   - 形式化梯度恒等式：nabla_theta(rho * k3) = rho * s_theta * ell_t
   - 当教师和学生一致时（ell_t = 0），SDL 梯度恰好为零。

3. 动态教师同步（Dynamic Teacher Synchronization）：
   - 冻结教师会随训练进展落后于改进中的学生。
   - Skill-SD 在每次迭代时从最新学生检查点同步教师。
   - 系统消融证明：教师拥有的 off-policy rollouts 在训练中期崩溃，
     冻结教师收敛到较低平台。

4. UCB 索引的技能缓冲区：
   - 每个任务选择最高 UCB 分数的技能。
   - score(e) = r_bar(e) + c * sqrt(ln(N_ucb) / n(e))
   - 新生成的技能总是先被尝试。

--------------------------------------------------------------------------------
【实验方法与过程】
--------------------------------------------------------------------------------
评估环境：
  - AppWorld：多应用 API 基准，9 个模拟消费应用，457 个 API，57 个开发集任务
  - Sokoban：经典推箱子规划基准，6x6 房间，2 个箱子，64 个测试关卡

基线对比：
  - Vanilla GRPO：标准 GRPO，无技能提示或自蒸馏
  - Skill-Augmented GRPO：技能前置到 rollout 提示，无 SDL 损失
  - Vanilla OPD：在策略蒸馏，无 RL 奖励
  - Base Model：未微调的 Qwen3-4B-Instruct-2507

训练配置：
  - 基础模型：Qwen3-4B-Instruct-2507
  - 群组大小 G 用于 GRPO 采样
  - 非对称裁剪边界（DAPO 风格）防止过早熵坍塌
  - 辅助 LLM 仅用于将轨迹总结为技能，不参与优化目标

消融分析：
  - 学生拥有的 rollout 至关重要：教师拥有的 rollouts 在训练中崩溃
  - 动态同步优于冻结教师
  - 技能应引导教师而非学生
  - 纯自蒸馏（无奖励信号）失败
  - SDL 权重 lambda 控制 RL-蒸馏平衡

--------------------------------------------------------------------------------
【模型】
--------------------------------------------------------------------------------
- 基础模型：Qwen3-4B-Instruct-2507
- 辅助 LLM：用于轨迹→技能总结（不参与优化）
- RL 算法：GRPO + SDL 联合目标
- 训练目标：L_total = L_GRPO + lambda * L_SDL

--------------------------------------------------------------------------------
【Prompt 设计】
--------------------------------------------------------------------------------
论文附录提供了详细 Prompt 模板（附录 C）：

1. 技能格式（Appendix A）：
   - 每个技能总结三个方面：
     e = (e_success, e_mistake, e_workflow)
   - 捕获：什么有效、什么失败、下次应遵循什么高层工作流。
   - 不存储完整轨迹作为技能（多轮任务有多种有效动作序列）。

2. 轨迹分析提示：指导将已完成轨迹总结为紧凑技能。

3. 学生提示：纯任务提示（无技能注入）。
4. 教师提示：任务提示 + 检索到的技能（S(x)）。

5. UCB 检索：每个任务选择单一最高分技能，轻量级检索。

核心训练循环：
  - 每个迭代：theta_old <- theta; 同步教师 theta_bar <- theta
  - Rollout：从学生采样 G 条轨迹，记录 log pi^stu
  - Re-score：计算当前学生 log pi^stu 和教师 log pi^tea
  - 更新：L = L_GRPO(r, A_hat) + lambda * L_SDL(rho^on, ell)
  - 异步：总结新轨迹为技能，更新缓冲区

--------------------------------------------------------------------------------
【关键实验数据】
--------------------------------------------------------------------------------
- AppWorld：64.9% 准确率（+56.1pp over base，+14.0pp over GRPO）
- Sokoban：62.5% 准确率（+50.0pp over base，+10.9pp over GRPO）
- Skill-Augmented GRPO 在两个基准上都不如 Vanilla GRPO（42.1% vs 50.9% on 
  AppWorld; 20.3% vs 51.6% on Sokoban）——直接将技能注入学生提示适得其反
- Vanilla OPD 是最弱的无奖励方法
- SDL 损失在训练中下降 59.3%，确认学生-教师分布收敛
